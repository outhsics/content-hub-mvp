## 🎯 使用建议

### Week 1: 熟悉系统
1. **本地测试**
   ```bash
   cd ~/content-hub-mvp/backend
   npm run test:daily
   ```

2. **查看生成质量**
   - 访问 http://localhost:3000
   - 检查每篇文章
   - 识别哪些风格表现好

3. **手动发布测试**
   - 选择最好的 5-10 篇
   - 发布到各平台
   - 观察反馈数据

### Week 2: 优化策略
1. **调整信源优先级**
   ```sql
   UPDATE sources SET priority = 9 WHERE name = '表现好的信源';
   ```

2. **优化 AI Prompt**
   - 在 `backend/src/services/ai/` 中修改 prompt
   - 测试不同表达方式
   - 记录哪些更有效

3. **发布策略**
   - 每天发布 10-15 篇（不是全部 30 篇）
   - 选择质量最高的
   - 错峰发布（避免平台检测）

### Week 3+: 规模化
1. **增加信源数量**
   ```sql
   INSERT INTO sources (name, source_type, url, priority) VALUES
   ('新信源', 'rss', 'https://...', 8);
   ```

2. **添加更多平台**
   - 百家号（需要认证）
   - 知乎（手动发布）
   - 微信公众号（需要认证）

3. **数据分析**
   - 记录每篇阅读量、点赞、评论
   - 计算每篇收益
   - 优化内容方向
